# MapPi3 V1.2.5 Live Map + Weather Pass

## Purpose

V1.2.5 starts addressing the real-world problem from phone testing: the Daily Exercise page needs a real map and current local conditions, not a stylized mock.

## Added

- Leaflet/OpenStreetMap component at `src/components/LiveLeafletMap.jsx`.
- Daily Exercise now renders a real OSM tile map.
- Daily Exercise traces points onto the Leaflet map.
- Map centers on Lewiston / Sabbatus line, Maine 04240 by default.
- Open-Meteo forecast fetch updates local temperature and humidity.
- Open-Meteo air-quality fetch updates US AQI when available.

## Still important

Phone browser GPS still generally requires HTTPS or localhost. The current public preview is plain HTTP by IP, so a phone may still reject live geolocation. When blocked, MapPi3 uses a simulator trace so the UI/path behavior can keep moving.

## Next options for true phone GPS

- Put this preview behind Vercel HTTPS.
- Use Tailscale Serve HTTPS if the phone is on Tailscale.
- Configure HTTPS reverse proxy for the VM.
- Later, package the Pi app with a secure local strategy or native wrapper.

## Data sources

- Map tiles: OpenStreetMap tile server for prototype only.
- Weather: Open-Meteo Forecast API.
- AQI: Open-Meteo Air Quality API.

For production/offline hiking, map tiles must be predownloaded as legal/offline route packs rather than depending on public live tile servers.
