# MapPi3 V1.2.8 Phone Walk Saving + Tailscale Workflow

## What changed

MapPi3 is still phone-first. Raspberry Pi GPS/Sense HAT, Supabase, GitHub, and Vercel stay later. This pass makes the phone walk tracker save real local sessions instead of only holding the current trace.

## Walk saving

Daily Exercise now supports:

- Start walk
- Stop + Save
- Discard trace
- Saved walk history
- Saved distance estimate from GPS coordinates
- Duration
- GPS point count
- Source label: `phone GPS`, `simulator`, or `GPS permission/error`
- GPX export per saved walk
- Delete saved walk

Saved walks are stored in browser localStorage under:

```text
mappi3.walkHistory
```

This is local to the exact browser origin. If the phone switches from Localtunnel to Tailscale HTTPS or raw HTTP, the browser treats it as a different app storage bucket. Use one stable Tailscale HTTPS URL for testing.

## Phone GPS notes

Mobile browsers require a secure context for geolocation permissions. Use HTTPS through Tailscale Serve where possible. Raw `http://100.82.165.23:5173` may render the app but usually cannot use phone GPS.

While recording a walk, keep the phone screen awake. Browser/PWA background tracking can pause when the screen locks or the browser is backgrounded. Native/PWA service worker background tracking is a later phase and still platform-limited.

## Tailscale Serve target

The preferred local target is the Vite dev server:

```bash
npm run dev -- --host 0.0.0.0 --port 5173
tailscale serve --bg --https 8443 http://127.0.0.1:5173
```

Port `8443` avoids the VM's existing public nginx listener on `443`.

Current phone URL:

```text
https://terminal-vnic.tailac984b.ts.net:8443/
```

Open that Tailscale HTTPS URL from the phone on the same tailnet.
