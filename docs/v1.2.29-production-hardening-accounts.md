# MapPi3 V1.2.29 Production Hardening + Local Accounts

## Added
- Route results no longer auto-populate when search is empty. The app prompts the user to type in search first.
- Added Account tab with CAK3D, tiny-Z, and Guest.
- CAK3D/tiny-Z first setup uses code `0000`, then requires a verified password on that device.
- Guest uses no password and clears guest session data on app exit/switch.
- Added device name and device role memory in Account/Settings-backed local storage.
- Added route geometry quality labels so rough seed geometry is never presented as survey-grade trail data.
- Removed unused SVG fake map component and replaced remaining workflow wording with production-facing language.

## Remaining production blockers
- True production auth should move to Supabase Auth or another server-side auth provider. Local hashes are only a device gate.
- Real trail accuracy requires verified GPX/OSM/Supabase geometry for each trail; seed geometry remains labeled as rough.
- GitHub/Vercel/Supabase are not wired yet in this release.

## OSM geometry imported

Imported OpenStreetMap/Overpass trail geometry for priority Grafton/Route 26 area routes: Table Rock, Old Speck AT segment, Eyebrow/Old Speck area, Speck Pond Trail, and Mahoosuc/AT segment. These are labeled `OSM trail geometry — verify current conditions`; they are better than straight seed lines but still require current trail/access verification before field reliance.
