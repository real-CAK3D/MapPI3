# MapPi3 V1.2.4 Daily Exercise + Health Refactor

## Purpose

V1.2.4 responds to field-use corrections:

- The map should not dominate every app section.
- Daily walks need their own page separate from planned hikes.
- Health intake should be practical: searchable foods and 8 oz water increments.
- Height should be entered as feet + inches.
- Vitals need plain-language labels.
- Conditions should be live/local/route-based later, with Lewiston/Sabbatus 04240 as the current default context.

## Added / changed

- New `Daily Exercise` tab.
- Map moved out of the global layout and now appears only on:
  - `Navigate`
  - `Daily Exercise`
- Daily Exercise can start/stop a walk trace.
- Browser geolocation is attempted only when a secure browser context is available.
- HTTP preview falls back to a simulator because most phones block `navigator.geolocation` on raw HTTP IP URLs.
- Food intake is now a searchable/selectable food list.
- Food calories are accumulated from selected foods.
- Water uses `+ 8 oz` and `− 8 oz` buttons.
- Height is now feet + inches.
- Vitals are simplified:
  - heart rate = beats per minute
  - blood pressure top/bottom = cuff reading if available
  - pulse helper counts beats over seconds and calculates BPM
- Conditions default to Lewiston / Sabbatus line, Maine 04240 demo context.

## Important geolocation note

Live phone GPS in a browser generally requires HTTPS or localhost. The current preview URL is plain HTTP by IP, so phone GPS may be blocked by the browser. Real options later:

- Vercel HTTPS preview for phone GPS testing.
- Tailscale Serve HTTPS.
- Pi-hosted local app with proper secure context strategy.
- Native wrapper later if needed.

## Safety boundary

Health, calories, hydration, heart rate, and weather condition adjustments are estimates. MapPi3 must not diagnose health issues, replace medical tools, or claim exact muscle gain.
